# ADR 0036 — temp/ Rendering Aid Pattern for ICEA and Tech Spec

**Date:** 2026-06-17
**Status:** Accepted
**Plugin version:** ≥ 2.4.0

---

## Context

ICEA and Tech Spec documents generated by `icea-feature` are 150+ lines of
structured markdown with tables, nested AC lists, and sign-off sections.
Presenting them inline in the chat stream makes them difficult to read and
review — tables render poorly in chat, headings don't provide navigation,
and the document is mixed with Claude's commentary.

VS Code's built-in markdown preview panel renders tables, headings, and
lists correctly. The developer can open a file with `Ctrl+Shift+V` and
review it in a dedicated panel while keeping the chat on the other side.

---

## Decision

ICEA and Tech Spec drafts are written to a `temp/` folder at the project
root instead of being output inline in chat.

**File naming:**
```
temp/ADO-{ID}-icea.md
temp/ADO-{ID}-tech.md
```

**Lifecycle:**
1. Written immediately after drafting (no user approval required — see below)
2. Rewritten in place after each iterative change during review
3. Copied to the permanent `docs/` location on `SAVE ICEA` / `SAVE TECH`
4. Deleted after copy — temp is never the source of truth

**Plan stays inline** — the Plan (~20 lines) is short enough that chat is fine.

---

## TEMP_WRITE_EXEMPT Convention

Writing to `temp/` before explicit user approval would normally violate the
global write gate (`## 0. WRITE GATE`). A skill-scoped exemption is used
instead of relaxing the global gate.

Skills that write to `temp/` must declare a `## TEMP_WRITE_EXEMPT` block
specifying:
- Which file patterns are exempt (`temp/ADO-{ID}-*.md`)
- Why (rendering aid, not source of truth)
- Lifecycle (when files are deleted)

The global write gate is NOT modified. The exemption is visible, auditable,
and scoped to the declaring skill only.

---

## Consequences

**Positive:**
- 150+ line documents readable in VS Code preview with proper table rendering
- VS Code auto-refreshes preview on file change — no manual reload needed
- Chat stays clean: only gap lists, change instructions, and one-line confirmations
- Temp files are self-cleaning — no manual housekeeping required

**Negative / mitigations:**
- Developer must open the temp file manually the first time — Claude always
  provides the path and shortcut (`Ctrl+Shift+V`) in the notification message
- `temp/` must be in `.gitignore` — handled automatically by gitignore-sync
  (added to managed BASE entries in v2.4.0)
- Abandoned sessions leave temp files on disk — cleaned up by dream-sync

---

## Alternatives Rejected

**Inline chat output (prior behaviour):** 150+ lines unreadable, tables don't
render, document mixed with Claude commentary.

**Write gate carve-out in CLAUDE.md (global):** Punches a hole in the hardest
guardrail. Rejected in favour of the scoped TEMP_WRITE_EXEMPT pattern.

**Write to permanent location immediately, no temp:** Violates the draft→review→save
flow established in ADR 0034. Permanent files should only exist after explicit
developer approval.
