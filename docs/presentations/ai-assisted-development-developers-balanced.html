Three lessons behind every decision

💸

Stateless skills are expensive

Code review re-scanned all 50 files daily even when 2 changed. Shared infrastructure fixed it.

🗺️

Orientation beats source reads

Read a graph generated once, in ms — not the source, on every invocation.

⚖️

Governance must be auditable

Consent categories, business severity, a validator — because informal conventions drift.

The shared primitives layer reduced daily review token cost by 80–95% after the first baseline run.

Why we decided X over Y

Six decision stories — problem, decision, alternatives rejected, lesson.

The gate that blocks your own PRADR 0001

Problem

AI used inconsistently; rework dominated. Most blocked PRs traced to a thin or absent spec.

Decision

No code until an ICEA is on disk and approved. "The ICEA is a contract, not documentation."

Lesson

Incentive changes behavior: a bad ICEA blocks your own PR. One good spec prevents ~3 bounced PRs.

Rejected: prompt-for-spec · review-then-code · optional ICEA.

→

→
code
no ICEA
→
✗ no code

One good spec ≈ 3 PRs that don't bounce

One bug, three copies, unfixed foreverADR 0003

Problem

Three skills, three copies of every scan bug; a fix in one never propagated. Gate logic even diverged between pr-create and checkin.

Decision

Used by 2+ skills → promote to skills/shared/; local copies forbidden; validator enforces delegation.

Lesson

Define once, maintain once. New skills reuse flags, routing, consent, severity — cheap to add.

Rejected: copy-paste ownership · big shared library up-front · "just document it".

Why the graph isn't just a pretty fileADR 0038·0039·0041

Problem

Two overlapping maps (drift); relationships as prose (unqueryable); entry-point-only staleness; a referenced git hook that didn't exist.

Decision

Single orientation layer (0038); authoritative graph.json with typed edges + module fingerprints (0039); EXTRACTED edges by script (0041).

Lesson

One source of truth; correctness before richness; mechanical edge extraction belongs in scripts.

Rejected: keep both maps · gitignore the graph · model-extracted edges.

Approval that works across SlackADR 0031

Problem

Approve/implement assumed a live session; APPROVE after reopening did nothing — state lived in chat, not on disk.

Decision

The ICEA Status: line is the single source of workflow state; skills read from disk every invocation.

Lesson

Close/reopen freely; approval from email or Teams is actionable with one phrase.

Rejected: session-only state · dual truth chat+file.

Prompts propose, hooks disposeADR 0005·0009

Diagram 2 of 7
Problem

Every hard rule was a model instruction — probabilistic. Local hooks bypassable (--no-verify); only opt-in teams install them.

Decision

Three tiers; each rule at the lowest tier that can hold it. "Prompts propose, hooks dispose."

Lesson

A local bypass becomes a CI failure — telemetry, not a silent skip.

Rejected: all-prompts · all-hooks · trust developers.
A
Model instructions
Judgment — probabilistic.
B
PreToolUse hooks
Deterministic per tool call.
C
Git / CI server-side
Unbypassable — required build validation.

The business knows what "medium" meansADR 0015

Problem

A CVSS "medium" exposing immigration IDs or privileged matter isn't medium here. Generic scores don't know the business.

Decision

Seven triggers (B1–B7) escalate one band across all review skills; visible in the ledger; dismissible with justification.

Lesson

Domain context is non-optional; severity without it is folklore. Extensible — new categories follow B8+.

Rejected: per-skill hard-coded overrides · manual bumps · ignore context.

How it got here

1.31
Write gate + hierarchical ICEA — a moment to review every write; no more filename collisions.
2.0
Session-independent workflow — approve/implement/status from disk; action approvals from Teams.
2.1
Single-responsibility + draft-then-save — review plans before ICEA, ICEA before tech spec.
2.6
Knowledge graph + /graph-sync — stop re-reading the codebase every morning.
