# Migration 024 — v3.13.0

## Summary

Feature: `external_detected_stacks` — setup-init and setup-sync now collect external
repository paths, detect their stacks, and store the results in `dream-init-state.json`
alongside `detected_stacks`. `icea-feature` overlay selection reads both fields and
picks the correct Tech Spec overlay. Two new Tech Spec overlays are implemented:
`techspec-aspnet-api-angular.md` (Angular + .NET API, no BFF) and
`techspec-angular-nodejs.md` (Angular + Node.js BFF + optional .NET API).

Bug fix: `dotnet-framework` (hyphen) corrected to `dotnet_framework` (underscore)
in the `icea-feature` overlay selection table, matching the actual token values
written to `dream-init-state.json`.

## Change type

New feature + bug fix. Target projects should run `/setup-sync` to receive the new
state fields and access the updated overlay selection logic.

## Required action

Run `/setup-sync`. The post-sync Step 5b external check will:
- Seed `external_detected_stacks: []` and `external_stacks_prompted: false` if absent
- Run detection silently if `additionalDirectories` is already populated
- Ask the developer for external repo paths if `additionalDirectories` is empty and the user has not been prompted before

## Sync actions (setup-sync Step 4 applies these, then Step 5b handles detection)

1. Seed `external_detected_stacks: []` in `.claude/dream-init-state.json` if the key is absent.
2. Seed `external_stacks_prompted: false` if the key is absent.
3. Run Step 5b — External stack detection (field-state-triggered, see setup-sync SKILL.md).

## New fields in dream-init-state.json

| Field | Type | Default | Purpose |
|---|---|---|---|
| `external_detected_stacks` | `string[]` | `[]` | Stacks found in `additionalDirectories` repos |
| `external_stacks_prompted` | `boolean` | `false` | Set once user answers the external-repo question; prevents re-prompting |

Stack tokens: `dotnet`, `dotnet_framework`, `angular`, `nodejs`, `java`, `python`.
`dotnet` and `dotnet_framework` are mutually exclusive.

## New files (in plugin — accessed via $PLUGIN_DIR)

- `scripts/external-stack-detection.cjs` — zero-dependency Node.js script.
  Run via: `node "$PLUGIN_DIR/scripts/external-stack-detection.cjs"`
- `skills/icea-feature/references/techspec-aspnet-api-angular.md` — new overlay
- `skills/icea-feature/references/techspec-angular-nodejs.md` — new overlay
