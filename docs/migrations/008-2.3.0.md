# Migration 008 — 2.3.0 (dream-init CLAUDE.md fixes)

Summary: Fixed dream-init path resolution and section detection.
Two bugs: `!ls CLAUDE.md` resolved to plugin dir; only `# Dream`
marker checked. Now uses `./CLAUDE.md` and checks all 7 required sections.

## Changes applied by /dream-sync

### 1. No CLAUDE.md content changes
This migration fixes dream-init behaviour only. No new sections are
added to project CLAUDE.md files by this migration — existing projects
already have the sections from migration 007.

### 2. dream-init behaviour (automatic on next run)
Step 1 now uses `[ -f "./CLAUDE.md" ]` instead of `!ls CLAUDE.md`.
Step 5 now checks all 7 required section markers:
  - `# Plugin version:`
  - `## 0. WRITE GATE`
  - `## 0a. Keyword Handlers`
  - `## 0b. Shell & Git Configuration`
  - `## Data Access Convention`
  - `## Feature Gate`
  - `# Dream`

Missing sections are appended by reading from the installed plugin's
CLAUDE.md — content is never hardcoded inline.

### 3. No skill, stub, or hook changes
All command stubs, skills, and hooks unchanged from 2.2.0.
