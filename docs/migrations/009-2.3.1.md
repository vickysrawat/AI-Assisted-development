# Migration 009 — 2.3.1 (path detection fallback chain)

Summary: Fixed `where.exe`-only path detection in dream-init and dream-sync.
Added fallback chain (`where.exe` → `which` → known install paths).
Fixed dream-sync silently skipping resolution when `⚠ NOT DETECTED` was
already written. Both now strip NOT DETECTED back to placeholder before retry.

## Changes applied by /dream-sync

### 1. CLAUDE.md — re-resolve Shell & Git paths if NOT DETECTED

If `## 0b. Shell & Git Configuration` contains `⚠ NOT DETECTED`, attempt
re-resolution using the fallback chain:

```bash
# Strip NOT DETECTED back to placeholder
sed -i 's|⚠ NOT DETECTED — run where\.exe git and update manually|{GIT_PATH}|g' ./CLAUDE.md
sed -i 's|⚠ NOT DETECTED — run where\.exe bash and update manually|{BASH_PATH}|g' ./CLAUDE.md

# Fallback chain
GIT_PATH=$(where.exe git 2>/dev/null | head -1 | tr -d '\r\n')
[ -z "$GIT_PATH" ] && GIT_PATH=$(which git 2>/dev/null | tr -d '\r\n')
[ -z "$GIT_PATH" ] && GIT_PATH=$(ls "/mingw64/bin/git.exe" "/usr/bin/git" \
  "C:/Program Files/Git/bin/git.exe" \
  "C:/Program Files/Git/mingw64/bin/git.exe" 2>/dev/null | head -1)

BASH_PATH=$(where.exe bash 2>/dev/null | head -1 | tr -d '\r\n')
[ -z "$BASH_PATH" ] && BASH_PATH=$(which bash 2>/dev/null | tr -d '\r\n')
[ -z "$BASH_PATH" ] && BASH_PATH=$(ls "/usr/bin/bash" "/bin/bash" \
  "C:/Program Files/Git/bin/bash.exe" \
  "C:/Program Files/Git/usr/bin/bash.exe" 2>/dev/null | head -1)

[ -n "$GIT_PATH" ]  && sed -i "s|{GIT_PATH}|$GIT_PATH|g" ./CLAUDE.md
[ -n "$BASH_PATH" ] && sed -i "s|{BASH_PATH}|$BASH_PATH|g" ./CLAUDE.md
[ -z "$GIT_PATH" ]  && sed -i 's|{GIT_PATH}|⚠ NOT DETECTED — run where.exe git and update manually|g' ./CLAUDE.md
[ -z "$BASH_PATH" ] && sed -i 's|{BASH_PATH}|⚠ NOT DETECTED — run where.exe bash and update manually|g' ./CLAUDE.md
```

If paths were already resolved correctly (no NOT DETECTED, no placeholders),
skip this step — nothing to do.
