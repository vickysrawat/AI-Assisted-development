# Migration 016 — v3.6.1

## 1 — Remove legacy `skills/` directory from project root

A bootstrap change introduced and reverted during 3.6.x development incorrectly copied
plugin reference files to `skills/` at the project root (shared specs, SKILL.md files,
per-skill references). These files are no longer deployed there and are no longer
gitignored.

**Check:** if your project has a `skills/` directory at the root containing only plugin
files (not your own application source code), delete it:

```bash
rm -rf skills/
```

To confirm it only contains plugin files before deleting:
```bash
ls skills/
# Expect: shared/  code-review/  security/  graph-sync/  etc.
# Do NOT delete if this folder contains your application's source code.
```

---

## 2 — Resolve `.claude/skills/` conflicts from pre-`.hashes` deployments

v3.6.1 introduces `.claude/skills/.hashes` to track plugin-owned project skill files.
If an earlier plugin version deployed files to `.claude/skills/` before this mechanism
existed, dream-sync will warn:

```
⚠ project skill X.md predates .hashes and differs from plugin source — treating as customised.
  Run /dream-teardown --skills then /dream-sync to reset to plugin defaults.
```

**Option A — Reset to plugin defaults (recommended if you have not customised the files):**
```
/dream-teardown --skills
```
Then run `/dream-sync` to redeploy all plugin project skills fresh with `.hashes` tracking.

**Option B — Keep your customisations:**
No action needed for files you want to keep — they are now treated as developer-owned.
To accept the plugin version for a specific file, delete it from `.claude/skills/` and
run `/dream-sync`.

---

## 3 — Plugin path pointer

v3.6.1 adds `.claude/plugin-path.txt` — a single-line file containing the absolute path
to the installed plugin cache. Skills use this to locate shared reference files without
an inline `installed_plugins.json` registry lookup.

This file is written automatically by `dream-init` and `dream-sync`. No manual action
required. It is gitignored by the plugin's managed `.gitignore` block.
