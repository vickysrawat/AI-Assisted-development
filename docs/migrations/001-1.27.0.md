# Migration to 1.27.0
Summary: VCS-aware ignore file — TFVC repos need a `.tfignore`; the managed ignore block is re-derived per VCS.

## Sync actions (idempotent — never overwrite developer content)
- Run the VCS-aware ignore writer (see `skills/shared/vcs-detect-spec.md`): detect
  Git vs TFVC and ensure the managed block is present in the file the repo honours
  — `.gitignore` (Git) or `.tfignore` (TFVC). A project provisioned by <1.27.0 in a
  TFVC repo has no `.tfignore` at all, so generated files and the credential file
  were never protected — this is the load-bearing action for the upgrade.
- On TFVC, additionally check whether the credential file is already tracked and,
  if so, surface the `tf vc delete --keep-local` remediation (the ignore entry
  alone does not unprotect an already-tracked file).

## Notes
No state-file schema change in this release; `dream_init_plugin_version` is simply
re-stamped to 1.27.0 at the end of the sync. Only the ignore file is version-
sensitive here. Projects already on Git see no functional change beyond the
re-stamp — the managed `.gitignore` block is unchanged from 1.26.0.
