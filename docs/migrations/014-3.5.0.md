# Migration to 3.5.0
Summary: `EXTRACTED` graph edges are now derived deterministically by
`scripts/graph-extract-edges.js` (import parsing) instead of by the model. Existing graphs
gain parser-true edges on the next `/graph-sync`. ADR 0041 (extends ADR 0039).

## Sync actions (idempotent — never overwrite developer content)
- On the next `/graph-sync`, Step 8a runs `node "$PLUGIN_DIR/scripts/graph-extract-edges.js"`,
  which rewrites the `EXTRACTED` edges in `.claude/graph/graph.json` from actual source. This
  may change existing `EXTRACTED` edges (now parser-true) and re-serialize the file in the
  canonical format — a one-time diff; deterministic thereafter.
- Model-authored `INFERRED`/`AMBIGUOUS` edges are **preserved** (upgraded to `EXTRACTED` only
  when source confirms the same pair, keeping their `type`/`reason`). Stale/dangling edges drop.
- No new hooks, rules, or state files. `nodes` and `fingerprint`s are untouched by the extractor
  (fingerprints still come from the bash `graph_module_fingerprint` helper).

## Notes
- The extractor is Node-stdlib only and offline; it parses imports locally and never surfaces
  raw source to the model — consistent with the client-internal / no-external-connectivity posture.
- Nothing to run manually: this is skill-logic + a helper script, picked up on plugin update.
  A `/graph-sync` is the only action, and it is already the recommended post-upgrade step.
- `tests/validate.py` check 9 (unique ids, no dangling edges, index/detail projection agreement)
  continues to hold — the extractor never emits dangling edges.
