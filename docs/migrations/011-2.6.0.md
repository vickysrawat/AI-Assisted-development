# Migration 011 — 2.6.0 (codebase knowledge graph + output token savings)

Summary: Introduces a codebase knowledge graph (`.claude/graph/`) with an
always-loaded index and per-module conditional detail files. Adds `/graph-sync`
for incremental refresh, git hook staleness detection, and session-start
notification. Also updates the write gate to show diffs instead of full files,
and adds output style instructions to `settings.json` to reduce output tokens.

## Changes applied by /dream-sync

### 1. New command stub — graph-sync

```bash
cp "$PLUGIN_STUBS/graph-sync.md" ".claude/commands/graph-sync.md" 2>/dev/null
```

Makes `/graph-sync` available in the project.

### 2. CLAUDE.md — Write Gate (diff-only display)

Check for the old "Show the full intended content" write gate instruction:

```bash
grep -q "Show the full intended content" ./CLAUDE.md
```

If found, update the write gate step to diff-only display:

> Old: "Show the full intended content (or a clear summary if very large)"
> New: "For modifications: show a unified diff (changed lines + 3 lines context). For new files: show full content. Never re-output unchanged lines."

Copy the updated `## 0. WRITE GATE` section from the installed plugin's CLAUDE.md.

### 3. .claude/settings.json — output style instructions

Add `customInstructions` to `.claude/settings.json` to suppress response
preambles and prevent full-file echo when writing existing files.

Merge this entry (never overwrite existing keys):
```json
{
  "customInstructions": "Response style: suppress preambles and plan-restatement before tool calls. When writing to existing files, show only a unified diff (changed lines + 3 lines of context) rather than the full file content. When writing new files, show the full content. Never echo generated file content to chat if the content is also being written to disk."
}
```

### 4. .gitignore — add .claude/graph/ to managed block

Run `/gitignore-sync` to add the knowledge graph to the managed block.
Or add manually inside the managed block:
```
.claude/graph/
```

### 5. Git hooks — stale-detection (post-merge + post-checkout)

Deploy the graph stale-detection hook:
```bash
cp "$PLUGIN_HOOKS/graph-stale-detect.sh" .claude/hooks/
chmod +x .claude/hooks/graph-stale-detect.sh
[ -f .git/hooks/post-merge ] && cp .git/hooks/post-merge .git/hooks/post-merge.backup
cp .claude/hooks/graph-stale-detect.sh .git/hooks/post-merge
chmod +x .git/hooks/post-merge
```

For post-checkout, append without overwriting:
```bash
grep -q "graph-stale-detect" .git/hooks/post-checkout 2>/dev/null || \
  echo "bash .claude/hooks/graph-stale-detect.sh" >> .git/hooks/post-checkout
chmod +x .git/hooks/post-checkout 2>/dev/null || true
```

## What is NOT auto-applied

- **Knowledge graph generation**: requires running `/dream-init` (fresh projects)
  or manually creating `.claude/graph/` following the schemas in `skills/shared/`.
  `/graph-sync` only refreshes an existing graph — it cannot create one from scratch.

- **Effort defaults**: documented in `skills/shared/model-routing-spec.md`.
  No file changes required — skills enforce their own defaults.
