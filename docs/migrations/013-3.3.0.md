# Migration to 3.3.0
Summary: CLAUDE.md context footprint slimmed. The plugin's injected governance sections are
compressed (operative rules unchanged); reference detail moved to `skills/shared/write-gate-spec.md`
and `skills/shared/dream-reference.md`. `# Dream` moved to the end of CLAUDE.md, fixing a
latent injector duplication. A governance-aware CLAUDE.md size advisory is added.

## Sync actions (idempotent — never overwrite developer content)
- CLAUDE.md governance refresh: `dream-init`/`dream-sync` stale-detection now also replaces the
  pre-3.3.0 **verbose** `## 0. WRITE GATE` and `## 0a. Keyword Handlers` with the slim versions.
  Both are H2 sections, so replacement is bounded to the section — neighbours are untouched.
- The slim `# Dream`, `## 3. DESIGN PHILOSOPHY` (now a pointer to `rules/project-rules.md`), and
  `## 4. MODEL ROUTING` (now a pointer to `skills/shared/model-routing-spec.md`) apply to newly
  provisioned projects. Existing projects keep their current copies unless re-provisioned; this is
  cosmetic (token footprint) only — no behaviour change.
- New shared specs deployed: `write-gate-spec`, `dream-reference` (added to `components.shared`).
- No new hooks or rule files.

## Notes
- No operative governance changed: the WRITE GATE rule, the full Keyword Handlers table, the
  Feature Gate, Shell/Git config, and the Data Access convention all remain in CLAUDE.md
  (always-active). Only supporting detail/rationale moved to referenced specs.
- The Feature Gate is additionally pre-loaded by `session-start` from its own embedded copy, so it
  is unaffected by CLAUDE.md compression.
- After sync, `/dream-health` shows a CLAUDE.md size card; `/dream-init` prints a one-time size
  advisory when a project's CLAUDE.md exceeds the budget with meaningful movable content.
