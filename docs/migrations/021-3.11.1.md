# Migration to 3.11.1
Summary: Fix CLAUDE.md managed-section duplication — `extractSection` now matches the section separator under CRLF, so `setup-sync` no longer drags the template tail into each append.

## Sync actions (idempotent — never overwrite developer content)
- None automated. The fix is entirely in plugin code (`scripts/setup-init-bootstrap.cjs`,
  loaded from the plugin dir, not carried on the project). Re-running `/setup-sync` under
  3.11.1 now behaves correctly: a CLAUDE.md whose 7 managed section headers are each
  present once takes the "all sections present" path and appends nothing.

## Notes
**Root cause.** `extractSection()` located a section's end with an LF-only `\n---\n`
separator. The plugin's own `CLAUDE.md` template ships with CRLF endings, so `\n---\n`
never matched (`\r\n---\r\n`); on failure the function returned everything from the
section header to end-of-file. Each managed section that was (re-)appended therefore
dragged the entire template tail with it — which is why sections nearest the end of the
template (Feature Gate, Data Access Convention) multiplied the most across repeated runs.
The section **detection** regexes were never at fault (they use `^`/`/m` and work under
CRLF); only the extraction did.

**Fix.** The separator match is now CRLF-tolerant (`/\r?\n---\r?\n/`), and `stepClaudeMd`
normalizes the template to LF immediately after reading it (defense-in-depth). A newly
created CLAUDE.md (Case A) is therefore written with LF endings.

**Healing an already-duplicated CLAUDE.md.** The fix stops further growth but does **not**
auto-collapse existing duplicates (the "all sections present" path only stamps the version
line — it will not rewrite a developer's file, to avoid losing project content interleaved
between managed sections). To clean a bloated file, do it in two steps:
1. `/setup-teardown` — its `stripPluginSections` (CRLF-normalizes, splits on `---`) removes
   every managed plugin section.
2. `/setup-sync` — re-appends exactly one clean copy of each of the 7 sections.
Review the result before committing; re-add any project-specific prose that lived between
managed sections.
